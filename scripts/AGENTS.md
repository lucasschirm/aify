# scripts

Build-time scripts (not shipped as runtime code; run via package scripts).

| File | Purpose |
|------|---------|
| `build-template-db.ts` | `buildTemplateDb(targetPath?)` — builds an EMPTY `templates/template_db.sqlite3` from the Sequelize models. Run with `pnpm build:template-db`. The file ships in the npm package (`files` allowlist, TASK_030) and seeds `~/.aify/aifydb.sqlite3` on first use. |
| `build-base-config.ts` | One-off baseline generator: `buildBaseConfig(csvContent)` (pure) + `writeBaseConfig(csvPath?, outputPath?)` reads `reference_docs/plans/sys_dictionary.csv` and emits `src/config/base.json` in the `TrackConfig` shape (spec OS-14). One `column_types` entry per distinct CSV `internal_type` (8 keys: `script`, `script_plain`, `script_server_side`, `json`, `css`, `html`, `html_template`, `html_script`). **Not a build step** — run once via `pnpm tsx scripts/build-base-config.ts` to (re)generate the committed baseline; idempotent. |

The template DB must match the models. CI (TASK_031) rebuilds it and fails on drift.

`build-base-config.ts` is intentionally not wired into `pnpm build` or CI — `src/config/base.json` is a committed baseline regenerated by hand when the source CSV changes.
